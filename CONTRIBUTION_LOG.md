## Branch and Workflow Summary

The project used a shared GitHub repository with multiple branches for development.

Main branches and work areas included:

- `feature-product` for product model work.
- `feature-services` for service logic.
- `feature-menu` for the console menu and user interaction.
- `feature-validation` for validation and exception handling.
- `final-integration` for combining and finalizing the complete Milestone 4 submission.

The final project was completed on the `final-integration` branch before being prepared for merging into `main`.
